## Streamlit

- **说明**：Streamlit是一个开源的python库，专为数据工程师及机器学习工程师设计，用来快速基于python代码构建交互式的web网站，且无需会前端技术


- **使用步骤**：
  1. 安装 `pip install streamlit`
  2. 引入 `import streamlit as st`
  3. 调用streamlit api来构建网页
  4. 运行：`streamlit run xxx.py`

- API 查询在官方文档的<a href="https://docs.streamlit.io/develop/api-reference">API Reference</a>处 

> Streamlit只适用于数据分析，人工智能学习可视化等领域制作网页
> 真正的商业业务网页依然需要正式的前端技术


- **代码示例--基于MiniMax**
```py
  # 初期化chat
  system_prompt = "你的名字叫Jack，是一名说话简洁的AI助理，你会尽量简单的回答任何问题"
  partner = {'role': 'system', 'content': system_prompt}
  if "messages" not in st.session_state:
      st.session_state.messages = [partner]

  # チャット履歴表示 -- 通过 st.session_state.messages 全局变量，缓存对话历史
  for message in st.session_state.messages:
      if message["role"] == "user":
          st.chat_message("user").write(message["content"])
      elif message["role"] == "assistant":
          st.chat_message("assistant").write(message["content"])
      else: # do nothing
          pass

  # chat input
  prompt = st.chat_input("メッセージを入力してください")
  if prompt:
      # ユーザー入力
      st.chat_message("user").write(prompt)
      print(f"----> 调用LLM，prompt：{prompt}")
      # ユーザーメッセージ保存
      user = {'role': 'user', 'content': prompt}
      st.session_state.messages.append(user)

      # LLM呼び出し
      response = client.chat.completions.create(
          model="MiniMax-M2.7",
          messages=st.session_state.messages,
          stream=True,
          extra_body={"reasoning_split": True}
      )

      # LLM返事 (流式输出的解析方式)
      with st.chat_message("assistant"):  # 创建一个消息框，用于显示LLM的回复，消息框内部程序执行完毕，消息框会自动关闭
          placeholder = st.empty()        # 创建一个占位符，用于显示实时的LLM回复
          reply = ""
          for chunk in response:
              if chunk.choices[0].delta.content is not None:
                  reply += chunk.choices[0].delta.content
                  placeholder.write(reply)
      print(f"----> LLM回复，reply：{reply}")

      # # LLM返事 (非流式输出的解析方式)
      # reply = response.choices[0].message.content
      # st.chat_message("assistant").write(reply)
      # print(f"----> LLM回复，reply：{reply}")

      # LLM返事保存
      assistant = {'role': 'assistant', 'content': reply}
      st.session_state.messages.append(assistant)
```

> `st.session_state`是`streamlit`自带的全局状态，通过向里面创建键值对，然后再调用来做到随时取出缓存并使用的功能
> 示例代码中的`if "messages" not in st.session_state:`就是来创建一个`messages键`，并加入值的过程


> 推理式ai会在回复的过程中进行推理。
> 该推理过程会与assistant的常规回复分开保存 (`"reasoning_split": True`时)
>
> 推理是否也要存入缓存？
> 推理主要服务于 `function calling`的调试，让ai更精准的行动
> 这里用于练习的普通聊天模型，不需要存储推理


> **`with 语法`**：
>   进入一个临时环境，执行代码，执行完毕后自动收尾
> 
>   `with st.chat_message("assistant"):`
>       `st.write("你好")`
>       `st.write("我是Jack")`
>
>   创建一个 assistant消息框，在该消息框内输出内容，然后关闭消息框
