### 环境变量 .env 文件的操作

- 在python项目目录下创建 `.env`文件，并配置好必要的环境变量
- 通过`pathlib`的`Path.(__file__).resolve`，或者 `os.path`等方式获取到`.env`的位置
- 安装`dotenv库`，引入`load_dotenv`
- 写下加载环境变量的代码：`load_dotenv(.env的路径)`
- 获取环境变量：`os.getenv(变量名)`

**示例**

```py 
    from dotenv import load_dotenv
    # 加载变量文件
    load_dotenv(""".env的路径""")
    # 获取变量
    API_KEY = os.getenv("API_KEY")
    BASE_URL = os.getenv("BASE_URL")
    MODEL_NAME = os.getenv("MODEL_NAME", "MiniMax-M2.7")
```

### 文件路径操作库 -- `pathlib`

传统的python路径是用`os.path`来进行操作的，`pathlib`提供了一个更动态的路径操作库，方便路径相关代码书写

- `Path(__file__).resolve()` -- 用于获取代码所在文件的绝对路径，加上`.parent`便能获取父级目录
- 基于此，便可以对路径进行拼接或定位

**示例**：
- 假如文件路径为：`C:\AI-Partner\program\chat_service.py`

```py
    from pathlib import Path

    BASE_DIR = Path(__file__).resolve().parent      # program/  -- 获取代码所在文件的父级路径，以此为基底
    PROJECT_DIR = BASE_DIR.parent                   # AI-Partner/
    RESOURCE_DIR = PROJECT_DIR / "resources"        # AI-Partner/resources/
    SESSION_DIR = BASE_DIR / "session_data"         # program/session_data/
```
